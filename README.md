# منصة صيف الابتكار 2026

منصة واحدة تغطي: تسجيل المشاركين، حضور يومي عبر QR، وإصدار/التحقق من الشهادات — بنفس هوية الفلاير (كحلي/ذهبي/كريمي).

## الملفات
| الملف | الوظيفة | من يستخدمه |
|---|---|---|
| `index.html` | نموذج التسجيل العام + توليد رمز QR للمشارك | الجمهور |
| `checkin.html` | مسح رمز QR وتسجيل حضور اليوم (1/2/3) | فريق الإدارة (بعد تسجيل الدخول) |
| `admin.html` | لوحة تحكم: إحصاءات، جدول المشاركين، استيراد/تصدير Excel، إصدار شهادات | فريق الإدارة |
| `certificate.html` | عرض/تحميل/التحقق من الشهادة عبر كود الشهادة | الجمهور (المشارك) |
| `config.js` | إعدادات الاتصال بـ Supabase (رابط + مفتاح) | مشترك بين كل الصفحات |
| `schema.sql` | بنية قاعدة البيانات والصلاحيات | يُنفَّذ مرة واحدة في Supabase |

## خطوات التشغيل

### 1) إنشاء مشروع Supabase
1. أنشئ مشروعًا جديدًا على [supabase.com](https://supabase.com).
2. من **SQL Editor** الصق محتوى `schema.sql` بالكامل ونفّذه (يُنشئ الجداول، المسارات الستة تلقائيًا، والصلاحيات).
3. من **Authentication → Users** أضف حساب/حسابات لفريق الإدارة (بريد + كلمة مرور) — هذا ما سيُستخدم للدخول في `admin.html` و`checkin.html`.
4. من **Project Settings → API** انسخ:
   - `Project URL`
   - `anon public key`

### 2) ربط المشروع
افتح `config.js` وضع القيمتين مكان:
```js
const SUPABASE_URL = "https://YOUR-PROJECT.supabase.co";
const SUPABASE_ANON_KEY = "YOUR-ANON-PUBLIC-KEY";
```

### 3) الرفع على GitHub ثم Vercel (نفس الأسلوب المعتاد)
```bash
git init
git add .
git commit -m "منصة صيف الابتكار 2026"
git remote add origin <رابط المستودع>
git push -u origin main
```
ثم من Vercel: **New Project → استورد المستودع → Deploy** (لا حاجة لأي إعدادات بناء، الملفات ثابتة).

### 4) الروابط بعد النشر
- `https://your-domain/index.html` → رابط التسجيل (شاركه مع المشاركين)
- `https://your-domain/checkin.html` → صفحة تسجيل الحضور (على جهاز/تابلت في المكان)
- `https://your-domain/admin.html` → لوحة التحكم
- `https://your-domain/certificate.html?code=INV-2026-XXXX` → رابط الشهادة (يمكن وضعه داخل رمز QR ثانٍ يُرسل للمشارك بعد إصدار شهادته)

## سير العمل المقترح
1. المشارك يسجّل من `index.html` ويختار مسارًا واحدًا → يحصل على رمز QR خاص به (يُنصح بإرساله له عبر البريد أيضًا لاحقًا إن رغبت).
2. عند حضوره كل يوم، يُمسح رمزه من `checkin.html` فيُسجَّل حضور ذلك اليوم تلقائيًا (يتم تمييز يوم اليوم الحالي).
3. من `admin.html` يمكن متابعة نسب الحضور لكل مسار لحظيًا، واستيراد قوائم جاهزة من Excel، وتصدير البيانات في أي وقت.
4. بعد انتهاء المسار، زر **"إصدار شهادات لمن أتمّ الحضور"** في `admin.html` يصدر تلقائيًا شهادة لكل من حضر الأيام الثلاثة، وينشئ كودًا فريدًا لكل شهادة.
5. رابط شهادة كل مشارك متاح من نفس الجدول (زر "عرض") — يمكن تحويله لرمز QR وإرساله للمشارك ليحمّل شهادته كـ PDF أو ليتحقق أي طرف ثالث من صحتها.

## تنسيق ملف الاستيراد (Excel)
أعمدة الملف المطلوبة (بالعربي أو الإنجليزي):
```
الاسم | البريد | الجوال | الجهة | رمز_المسار
```
قيم `رمز_المسار` المسموحة: `biz`, `agents`, `coding`, `elearn`, `emerging`, `digital` (معرّفة في `schema.sql`).

## ملاحظات أمنية
- التسجيل العام مفتوح للجميع (Insert فقط)، بينما القراءة/التعديل محصورة على حسابات فريق الإدارة المسجَّلة عبر Supabase Auth.
- صفحة الشهادة العامة تستخدم دالة قاعدة بيانات آمنة (`get_certificate`) لا تكشف أي بيانات غير اسم المشارك والمسار والتاريخ.
- يمكن لاحقًا إضافة تحقق بريد إلكتروني (OTP) قبل التسجيل إذا رغبت بمنع التسجيلات الوهمية — أخبرني إن احتجت ذلك.
